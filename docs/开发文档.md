### 2.2 Agent 装配（`services/agent_service.py`）

- `create_agent()`：
  1. `model = create_llm()`（按 `LLM_PROVIDER` 选 volcano / siliconflow）。
  2. `tools = [...]`：**把所有工具注册到同一个 Agent**（见 2.3 清单）。
  3. `full_prompt = get_full_prompt(tools_list_text, skills_context, workspace_context)`。
  4. `agent = create_react_agent(name=..., model=model, tools=tools, prompt=full_prompt)`。
- `process_chat_stream(messages, session_id)`：每次请求新建 agent + `StreamProcessor`，`async for event in processor.process_stream(...)` 逐条 `yield` SSE 字符串。客户端断开（GeneratorExit/ConnectionError）被静默处理。

> **教学要点**：5 个工作台共用这一个 Agent。前端不同工作台只是注入不同 `systemHint` 文本拼到用户消息前，"该调哪个工具"仍由真实 LLM 决定。

### 2.3 已注册工具（节选自 `create_agent()`）

| 工具 | 文件 | 能力 |
|---|---|---|
| `generate_volcano_image` / `edit_volcano_image` | `tools/volcano_image_generation.py` | 火山 Seedream 文生图 / 改图 |
| `generate_3d_model` | `tools/model_3d_generation.py` | 腾讯混元 文/图生 3D |
| `generate_volcano_video` | `tools/volcano_video_generation.py` | 火山 Seedance 视频（异步） |
| `concatenate_videos` | `tools/video_concatenation.py` | 视频拼接（moviepy） |
| `detect_face` / `generate_virtual_anchor` | `tools/virtual_anchor_generation.py` | 数字人（未在本平台开放） |
| `qwen_voice_design` / `qwen_voice_cloning` | `tools/qwen_tts.py` | DashScope 音色设计 / 复刻 |
| `concatenate_audio` / `select_bgm` / `mix_audio_with_bgm` | `tools/audio_mixing.py` | 音频拼接/选 BGM/混音 |
| `qwen_omni_understand` | `tools/qwen_omni_understanding.py` | Qwen3-Omni 多模态理解 |
| `read_skill_file` / `list_skill_dir` / `init_skill` / `write_skill_file` / `delete_skill_file` | `tools/skill_tools.py` | Skill 读/写/创建 |
| `write_memory` | `tools/workspace_tools.py` | 写工作空间记忆 |

各工具内部用 `requests` 调对应厂商 base_url（来自 `.env`），把返回的媒体 URL `requests.get` 下载到 `storage/<类型>/`，并返回结构化结果（含 `image_url` / `video_url` / `audio_url` / `model_url` 等）。**mock 模式下 base_url 指向 `:8900`，下载链路完全不变。**

### 2.4 SSE 事件协议（`stream_processor.py` 产出，前端 `agentRuntime.ts` 消费）

每行格式 `data: <json>\n\n`，`data: [DONE]` 表示结束。事件类型：

| `type` | 字段 | 含义 |
|---|---|---|
| `delta` | `content` | 助手文本增量（逐字/逐块） |
| `skill_matched` | `skill_name` | 命中某个 Skill（前端渲染成思考链一条） |
| `tool_call` | `id`, `name`, `arguments` | Agent 决定调用某工具 |
| `tool_result` | `tool_call_id`, `content`(JSON 字符串) | 工具返回；`content` 里可能含 `image_url`/`video_url`/`audio_url`/`model_url`(+`preview_url`/`format`/`mtl_url`/`texture_url`)/`prompt` |
| `error` | `error` | 出错信息 |

复用的原有组件：`src/components/Model3DViewer.tsx`（OBJ/GLB WebGL 预览）等。

### 3.2 路由

`App.tsx` 用 URL query 决定渲染：`?view=image|model3d|video|podcast|chat`，无 `view`→首页；`?page=settings`→设置。切换通过 `pushState` + `popstate` 事件。**无前端路由库**，纯 query 参数。

### 3.3 `agentRuntime.ts`（前后端桥接，必读）

`useAgentRuntime({ canvasId, systemHint, onMedia, onModel })` 返回 `useLocalRuntime(adapter)`。adapter 的 `run()`：

1. 把历史 messages 映射成 `{role, content}`；对历史 assistant 消息，把其 tool-call 结果里的 `image_url/audio_url` 追加进文本，便于多轮上下文。
2. `messageToSend = systemHint ? systemHint + "\n\n" + userText : userText`。
3. `fetch('/api/chat', { method:'POST', body:{message, messages, canvas_id}, signal: abortSignal })`。
4. 读取 SSE 流，按事件类型构建 assistant-ui 的 `parts`（`text` / `reasoning` / `tool-call`），每次 `yield snapshot()` 触发 UI 增量渲染。
5. 收到 `tool_result` 时，按结果字段回调 `onMedia({kind:'image'|'video'|'audio', url, prompt})` 或 `onModel({modelUrl, previewUrl, format, mtlUrl, textureUrl})`——这就是右侧预览面板更新的入口。

> Vite dev 把 `/api` 和 `/storage` 代理到 `http://localhost:8000`（见 `frontend/vite.config.ts`）。所以前端只管打 `/api/chat`，无需关心后端端口。

### 3.4 视觉规范（避免回到"AI 味"）

- 配色：暖近黑底 + 灰白文字，**无蓝/紫/绿**强调色；高对比中性（白底深字）做按钮/用户气泡。
- 分区：**不用 1px 硬分割线**，改"圆角浮层面板（~18px 圆角）+ 留白"软分区（参考 Coze）。
- 左栏：单一 Agent 切换栏（头像+名称+副标题），图标竖列统一左边缘与尺寸；不放"xxx 平台"品牌块。

## 4. 模拟厂商 API（`mock-api/`）

独立 FastAPI（`server.py`），与主项目隔离（独立 venv/端口）。详尽说明见 [`mock-api/README.md`](../mock-api/README.md)。要点：

- `/files/...` 静态托管 `assets/` 下素材；各业务端点返回的 URL 都指向这里。
- 复刻端点（字段对齐官方文档与后端工具契约）：

| 厂商/能力 | 端点 | 模式 |
|---|---|---|
| 火山 文生图/改图 | `POST /api/v3/images/generations` | 同步，`data[].url` |
| 火山 视频 | `POST …/contents/generations/tasks` + `GET …/tasks/{id}` | 异步，`id`→`status`/`content.video_url` |
| 腾讯 3D | `POST /v1/ai3d/submit` + `POST /v1/ai3d/query` | 异步，`Response.JobId`→`Status`/`ResultFile3Ds[]` |
| DashScope TTS | `…/audio/tts/customization` + `…/multimodal-generation/generation` | 同步，`output.voice`/`output.audio.url` |
| DashScope Omni | `POST /compatible-mode/v1/chat/completions` | OpenAI 兼容，同步/流式 |

- 异步任务：内存表记录提交时间，经过 `MOCK_TASK_DELAY`（默认 6s）后由 `running/RUN`→`succeeded/DONE`，模拟真实轮询。
- 素材轮换：每次生成 `next(itertools.cycle(...))` 换一个文件，便于测试多产物累积。

## 5. 如何扩展（HOW-TO）

### 5.1 新增一个后端工具
1. 在 `backend/app/tools/` 新建 `xxx.py`，用 `@tool` 定义函数，内部调厂商 API → 下载落地 `storage/` → 返回含媒体 URL 的 dict。
2. 在 `agent_service.py` 的 `create_agent()` 里 `from app.tools.xxx import xxx_tool` 并加入 `tools=[...]`。
3. 若产物是新模态，前端在 `agentRuntime.ts` 的 `tool_result` 分支里加字段识别 + 回调。

### 5.2 新增一个前端工作台
1. 仿照 `ImageStudio.tsx` 写一个 `XxxStudio.tsx`：定义 `HINT`（systemHint），`useThreadScoped` 收集产物，`useAgentRuntime({systemHint, onMedia/onModel})`，右侧渲染你的预览组件。
2. 在 `nav.tsx` 的 `TrainingView` 加类型，`App.tsx` 的 `VALID_VIEWS` + 渲染分支加入，`CourseSidebar.tsx` 加一行 Agent。

### 5.3 新增/修改一个 mock 端点
1. 在 `mock-api/server.py` 加一个 `@app.post(...)`，**严格对齐目标厂商官方文档的请求/响应字段**。
2. 需要返回媒体就用 `file_url(sub, name)` 指向 `assets/`；异步任务参考 `_video_tasks` / `_ai3d_jobs` 的内存表写法。
3. 在 `backend/.env` 把对应 `*_BASE_URL` 指到 `:8900` 即可联调。

### 5.4 换 LLM / 接本地 Qwen3
- 换云 LLM：改 `.env` 的 `LLM_PROVIDER` + `OPENAI_*` / `VOLCANO_*`。
- 接本地 Qwen3（教学第九章）：用 vLLM 起一个 OpenAI 兼容服务，`LLM_PROVIDER=siliconflow`、`OPENAI_BASE_URL` 指向本地 vLLM 地址即可，**前后端代码无需改**。

## 6. 调试清单

- 后端是否在跑：`curl localhost:8000/health`。
- mock 是否在跑 + 加载了哪些素材：`curl localhost:8900/__mock/info`。
- 前端代理是否生效：浏览器 Network 看 `/api/chat` 是否 200 且持续有 SSE 数据。
- "点了没反应"：多半是点了第二次（命中"停止"按钮 abort）。看后端日志，通常请求其实跑完了。
- 媒体不显示但对话正常：检查 `tool_result.content` 里的 URL 字段名是否被 `agentRuntime.ts` 识别。
