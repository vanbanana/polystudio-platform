# 一、后端 API（:8000）

通用约定：

- Base URL：`http://localhost:8000`；REST 路由前缀 `/api`（在 `app/main.py` 注册）。
- 编码：请求/响应均为 `application/json; charset=utf-8`，除上传（`multipart/form-data`）与 `/api/chat`（响应为 `text/event-stream`）。
- CORS：`allow_origins=["*"]`（开发用；生产应收紧）。

---

## 1. POST /api/chat —— 对话（SSE 流式）

平台对话唯一入口。前端 5 个工作台都打这个端点，**共用同一个全能 Agent**。

**请求体**（`ChatRequest`，`app/routers/chat.py`）：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `message` | string | 是 | 本轮用户输入。前端会把工作台的 `systemHint` 拼在前面一起发。 |
| `messages` | array | 否 | 历史消息 `[{role, content}, ...]`（不含本轮）。 |
| `session_id` | string | 否 | 会话 ID（用于历史保存）。 |
| `canvas_id` | string | 否 | 画布/会话 ID。**若提供**，每个 SSE 事件会同时通过 `/ws/{canvas_id}` 广播。 |

```bash
curl -N -X POST http://localhost:8000/api/chat \
  -H 'Content-Type: application/json' \
  -H 'Accept: text/event-stream' \
  -d '{"message":"画一只在星空里的柴犬","messages":[]}'
```

**响应**：`text/event-stream`。每个事件一行 `data: <json>\n\n`，以 `data: [DONE]` 结束。事件类型见下节。

> ⚠️ 后端在每次请求新建一个 Agent 实例并流式处理；客户端断开（abort）属正常情况，后端会静默停止，不报错。

## 2. SSE 事件协议

由 `app/services/stream_processor.py` 产出，前端 `frontend/src/training/agentRuntime.ts` 消费。

| `type` | 关键字段 | 含义 | 前端如何处理 |
|---|---|---|---|
| `delta` | `content`(string) | 助手文本增量 | 追加到当前 text part，逐字渲染 |
| `skill_matched` | `skill_name`(string) | 命中某个 Skill | 渲染成思考链一条「命中技能：X」 |
| `tool_call` | `id`, `name`, `arguments`(object) | Agent 决定调用工具 | 新增一个 tool-call part（显示工具名+参数） |
| `tool_result` | `tool_call_id`, `content`(JSON 字符串) | 工具返回 | 解析 `content`，按媒体字段更新右侧预览（见第三节） |
| `error` | `error`(string) | 出错 | 在对话中追加「错误：X」 |
| `user_message` | `content`(string) | 用户消息回显（仅 WebSocket 广播用） | — |
| `done` | — | 结束（仅 WebSocket 广播时显式发送；HTTP 流用 `[DONE]`） | 收流结束 |

**事件示例（一次文生图）**：

```text
data: {"type": "skill_matched", "skill_name": "图片生成"}

data: {"type": "tool_call", "id": "call_abc", "name": "generate_volcano_image", "arguments": {"prompt": "星空里的柴犬"}}

data: {"type": "tool_result", "tool_call_id": "call_abc", "content": "{\"image_url\": \"/storage/images/volcano_xxx.jpg\", \"prompt\": \"星空里的柴犬\"}"}

data: {"type": "delta", "content": "已"}
data: {"type": "delta", "content": "为你生成"}

data: [DONE]
```

> `tool_result.content` 是**字符串化的 JSON**，需二次 `JSON.parse`。其内部字段名（`image_url`/`video_url`/`audio_url`/`model_url`…）决定前端把它渲染成哪种媒体。

## 3. WebSocket /ws/{canvas_id} —— 事件广播

`ws://localhost:8000/ws/{canvas_id}`（`app/main.py`）。订阅后，凡是带相同 `canvas_id` 的 `/api/chat` 请求，其每个 SSE 事件都会**实时广播**给所有订阅该 `canvas_id` 的 WebSocket 连接（包括从 Postman/外部触发的请求）。用于"外部驱动 → 画面同步"。本实训前端默认走 SSE，WS 为可选能力。

## 4. 画布 / 历史

| 方法 | 路径 | 说明 | 响应 |
|---|---|---|---|
| GET | `/api/canvases` | 获取所有画布/会话历史 | `[{id, name, messages, ...}]` |
| POST | `/api/canvases` | 新建或保存画布（直接存原始 JSON，不过滤 Excalidraw data） | 保存后的画布对象 |
| DELETE | `/api/canvases/{canvas_id}` | 删除画布 | `{"success": true}` |

历史持久化在 `backend/storage/chat_history.json`（由 `history_service` 管理）。

## 5. 文件上传

| 方法 | 路径 | 接收 | 落地目录 | 响应 |
|---|---|---|---|---|
| POST | `/api/upload-image` | `multipart/form-data` file（`image/*`） | `storage/images/` | `{"url": "/storage/images/upload_xxx.jpg", "filename": "..."}` |
| POST | `/api/upload-audio` | file（mp3/wav/m4a/aac/ogg/flac/wma） | `storage/audios/` | `{"url": "/storage/audios/...", "filename": "..."}` |
| POST | `/api/upload-video` | file（mp4/mov/avi/mkv/webm） | `storage/videos/` | `{"url": "/storage/videos/...", "filename": "..."}` |

文件名格式 `upload_<时间戳>_<8位uuid><ext>`。返回的是**相对 URL**，前端经 Vite 代理或后端 `/storage` 静态访问。

## 6. 设置

`app/routers/settings.py`，前缀 `/api`。敏感字段（含 `KEY`/`SECRET`/`TOKEN`/`PASSWORD`）在读取时脱敏。

### Skills（前端开关，按 category）
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/settings/skills` | `{"skills":[{id,name,description,enabled,category}]}`（已保存状态覆盖默认） |
| PUT | `/api/settings/skills` | body `{"skills":[{id,...,enabled}]}` → `{"ok":true}` |

### Installed Skills（基于 `skills/**/SKILL.md`）
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/settings/skills/installed` | `[{id,name,description,source("public"|"custom"),enabled}]` |
| PUT | `/api/settings/skills/installed/{skill_id}` | body `{"enabled":bool}`，启用/禁用 |
| GET | `/api/settings/skills/installed/{skill_id}/content` | `{"id","content"}`（SKILL.md 原文） |

### MCP / 环境变量 / 工作空间
| 方法 | 路径 | 说明 |
|---|---|---|
| GET / PUT | `/api/settings/mcp` | 读/写 `{"mcpServers": {...}}` |
| GET / PUT | `/api/settings/env` | 读/写分组环境变量（敏感值脱敏；写入 `backend/.env`） |
| GET | `/api/settings/workspace` | 列工作空间身份/记忆文件（AGENTS.md / IDENTITY.md / MEMORY.md 等） |
| GET / PUT | `/api/settings/workspace/{filename}` | 读/写指定工作空间文件 |

# 二、模拟厂商 API（:8900）

`mock-api/server.py`。**按各厂商官方文档字段复刻**请求/响应，但所有媒体 URL 都指向本服务 `/files/...`（静态托管 `mock-api/assets/`）。后端配 `.env` 把对应 `*_BASE_URL` 指到此服务即可。

通用：
- `GET /` → `{"service","status":"ok","public_url"}`
- `GET /__mock/info` → 已加载素材清单 + `task_delay_seconds` + 端点表（健康检查首选）
- `GET /files/{sub}/{name}` → 静态素材（images/videos/models/previews/audio）
- 异步任务（视频/3D）：提交后经 `MOCK_TASK_DELAY`（默认 6s）由 `running`→`succeeded`
- 每次生成轮换一个素材（`itertools.cycle`），便于测试多产物累积

### 2.1 火山 Seedream 文生图 / 改图（同步）

`POST /api/v3/images/generations`（后端 base 配 `…:8900/api/v3`）

请求（含 `image` 字段即视为改图）：
```json
{ "model": "seedream-4.5", "prompt": "星空里的柴犬", "n": 1, "size": "2048x2048" }
```
响应：
```json
{
  "model": "seedream-4.5",
  "created": 1750000000,
  "data": [{ "url": "http://localhost:8900/files/images/xxx.jpg", "size": "2048x2048" }],
  "usage": { "generated_images": 1, "output_tokens": 0, "total_tokens": 0 }
}
```

### 2.2 火山 Seedance 视频（异步：提交 + 轮询）

提交 `POST /api/v3/contents/generations/tasks`：
```json
// 请求
{ "model": "doubao-seedance-1-5-pro", "content": [...], "ratio": "16:9" }
// 响应
{ "id": "cgt-xxxxx", "model": "doubao-seedance-1-5-pro", "status": "queued" }
```
查询 `GET /api/v3/contents/generations/tasks/{id}`：
```json
// 未完成
{ "id": "cgt-xxxxx", "model": "...", "status": "running" }
// 完成
{ "id": "cgt-xxxxx", "model": "...", "status": "succeeded",
  "content": { "video_url": "http://localhost:8900/files/videos/xxx.mp4" },
  "usage": {...}, "created_at": ..., "updated_at": ... }
```

### 2.4 DashScope Qwen-TTS（同步）

音色定制 `POST /api/v1/services/audio/tts/customization`：
```json
// 请求（声音设计 model=qwen-voice-design / 声音复刻 model=qwen-voice-enrollment）
{ "model": "qwen-voice-design", "input": { "target_model": "...", "preview_text": "..." } }
// 响应（声音设计：含预览音 base64）
{ "output": { "voice": "vd-xxxxxxxx", "preview_audio": { "data": "<base64>" } },
  "usage": { "characters": 12 }, "request_id": "..." }
// 响应（声音复刻：仅返回音色 id）
{ "output": { "voice": "vc-xxxxxxxx" }, "usage": {}, "request_id": "..." }
```
语音合成 `POST /api/v1/services/aigc/multimodal-generation/generation`：
```json
// 请求
{ "input": { "voice": "vd-xxxxxxxx", "text": "今天聊聊咖啡文化……" } }
// 响应
{ "output": { "audio": { "url": "http://localhost:8900/files/audio/xxx.wav", "expires_at": 1750003600 } },
  "usage": { "characters": 20 }, "request_id": "..." }
```

### 2.5 DashScope Qwen-Omni 多模态理解（OpenAI 兼容，最小实现）

`POST /compatible-mode/v1/chat/completions`（支持 `stream: true`）。非流式返回标准 `chat.completion`；流式返回 `chat.completion.chunk` + `[DONE]`。核心 5 个工作台不依赖它。

# 三、工具返回字段 → 前端媒体渲染映射

`tool_result.content`（JSON）里出现以下字段时，`agentRuntime.ts` 会回调对应渲染：

| 工具结果字段 | 回调 | 前端渲染 |
|---|---|---|
| `image_url` | `onMedia({kind:'image', url, prompt})` | 右侧画廊缩略图 |
| `video_url` 或 `video_path` | `onMedia({kind:'video', url, prompt})` | 右侧视频播放器 |
| `audio_url` | `onMedia({kind:'audio', url, prompt})` | 播客单集卡片 + 波形 |
| `model_url`（+`preview_url`/`format`/`mtl_url`/`texture_url`） | `onModel({modelUrl, previewUrl, format, mtlUrl, textureUrl, prompt})` | 右侧 WebGL 3D 查看器（OBJ/GLB） |

> 新增模态时：在后端工具结果里加字段，并在 `agentRuntime.ts` 的 `tool_result` 分支里加识别 + 回调。
