## 1. 这个项目是什么 / 做了什么改造

原项目 **PolyStudio** 是一个「对话式画布」多模态创作平台：一个 LangGraph 全能 Agent + 一个 Excalidraw 大画布承载所有模态（文生图/3D/视频/TTS/混音/理解/虚拟人/Skill）。

本仓库在它之上做了**面向教学实训的改造**，目标是「不臃肿、聚焦、好看、好上手」：

1. **前端重构**（`frontend/src/training/`）：把"单画布跑全部"拆成 5 个**专门工作台**——文生图 / 3D 资产 / 短视频 / 播客 / Qwen3 对话；左侧是 Coze 风格的 Agent 切换栏 + 会话记录，中间对话，右侧媒体预览。对话/工具调用/思考链/输入框全部改用业界组件库 **@assistant-ui/react**，不再手写。视觉为中性纸感/暖近黑 + 圆角浮层分区，**无蓝/紫/绿强调色、无硬分割线**。
2. **后端**：**几乎未改动** Agent 与工具逻辑（教学价值就在这套编排内核）。仅通过 `backend/.env` 切换 provider 与 LLM。
3. **新增 `mock-api/`**：一个**独立、可随时启停**的「模拟厂商 API」服务，按官方文档字段复刻各厂商接口、返回本地公开素材，让全链路在不烧钱、不联外网厂商的前提下跑通，用于验证显示与功能对齐。

**未做**：数字人/虚拟主播（第五章，需本地 GPU + ComfyUI），首页里它显示"即将上线"。

## 2. 三套服务一览

| 服务 | 目录 | 端口 | 启动 | 日志 |
|---|---|---|---|---|
| 模拟厂商 API | `mock-api/` | 8900 | `./start.sh` / `./stop.sh` | `mock-api/mock-api.log` |
| 后端 Agent | `backend/` | 8000 | `python -m app.main`（reload）或 `./start.sh` | `backend/backend.log`、`backend/logs/` |
| 前端 dev | `frontend/` | 3000（被占用会自增到 3001/3002…） | `npm run dev` | 终端 |

依赖：Python 3.9+、Node.js 18+。后端与 mock-api 各自独立 venv（mock-api 的 `start.sh` 会自动建 `.venv` 并装依赖）。

启动顺序建议：**mock-api → backend → frontend**（后端启动时不强依赖 mock，但跑链路前要先起 mock）。

## 3. 把它跑起来（mock 模式，逐步）

```bash
# (1) 模拟厂商 API
cd mock-api && ./start.sh
curl http://localhost:8900/__mock/info        # 健康检查 + 查看已加载素材/端点

# (2) 后端
cd ../backend
cp env.example .env          # 然后按下面「关键 .env」编辑
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
python -m app.main           # http://localhost:8000 ，/health 应返回 ok

# (3) 前端
cd ../frontend
npm install
npm run dev                  # http://localhost:3000
```

### 关键 `.env`（`backend/.env`，**不进仓库**，已被 .gitignore 忽略）

```env
LLM_PROVIDER=siliconflow                 # 用 OpenAI 兼容通道接 DeepSeek
OPENAI_API_KEY=sk-xxxx                   # ←★ 必填：你的 DeepSeek key
OPENAI_BASE_URL=https://api.deepseek.com
MODEL_NAME=deepseek-v4-flash

VOLCANO_BASE_URL=http://localhost:8900/api/v3   # 火山图/视频 → mock
TENCENT_AI3D_BASE_URL=http://localhost:8900     # 腾讯 3D → mock
DASHSCOPE_BASE_URL=http://localhost:8900        # DashScope TTS/Omni → mock
MOCK_MODE=false                                 # 走真实 HTTP 链路，而非假数据绕过
```

> **唯一需要外部提供的东西 = 一个 OpenAI 兼容 LLM 的 key**（Agent 决策必须用真模型）。其余全部走本地 mock。
> ⚠️ 切勿把 key 提交进仓库。`.env` 已在 `.gitignore` 中。

### 切回真实厂商 API

把上面三个 `*_BASE_URL` 改回官方地址、填真实各厂商 key（见 `backend/env.example` 注释里的官方控制台链接），mock-api 关掉即可。互不影响。

## 4. 已验证可用（验收记录）

在 mock 模式 + 真实 DeepSeek 下，5 个工作台**全部走通真实 HTTP 链路并正确显示**：

| 工作台 | 链路 | 显示结果 |
|---|---|---|
| 文生图 | DeepSeek 决策 → 火山图 mock → 下载 → `/storage/images` | 右侧画廊出图，工具调用"生成图片 完成" |
| 3D 资产 | DeepSeek → 腾讯 3D mock（提交+轮询） → 下载 GLB | 右侧 WebGL 查看器渲染 GLB |
| 短视频 | DeepSeek → 火山视频 mock（提交+轮询） → 下载 | 右侧可播放成片 |
| 播客 | DeepSeek → DashScope TTS mock | 右侧单集卡片 + 波形播放器 |
| Qwen3 | DeepSeek 纯对话（无工具） | 全宽 Markdown 流式对话 |

## 5. 已知问题 / 注意事项

1. **建议词气泡是 `autoSend`**：点一次即发送。生成中原"发送"按钮位置会变成"停止"按钮，**再点一次会 abort 本次请求**（后端仍会跑完，但前端收不到后续 SSE）。这不是 bug，是交互细节——测试时**点一次后耐心等**即可。
2. **真实 LLM 有延迟**：DeepSeek 出结果约 8–90 秒（含思考链时更久）；视频/3D 是异步任务（mock 默认提交后约 6 秒由 `running`→`succeeded`，可用 `MOCK_TASK_DELAY` 调）。
3. **React StrictMode（dev）双挂载**：`frontend/src/main.tsx` 启用了 `StrictMode`，dev 下组件会双挂载。当前实现不受影响，但若改动 runtime/AbortController 生命周期需留意。
4. **mock 素材是轮换的**：每次生成换一个本地素材文件，用于测试"多图/多模型/多视频"累积；所以同一 prompt 多次生成图不一样是正常的。
5. **数字人未实现**：首页第五章卡片为 `disabled`（"即将上线"）。
6. **mock 素材随仓库提供**（约 60MB，`mock-api/assets/`），用 `git add -f` 越过了根 `.gitignore` 的媒体忽略规则；来源/重建见 `mock-api/download_assets.sh`。

## 6. 分支与来源（本仓库如何拼出来的）

本独立仓库的 `main` = 原 PolyStudio + 以下两条改造分支合并：

- **训练 UI**（原 PR：`vanbanana/polystudio-platform#1`，分支 `devin/...-training-platform-ui`）：`frontend/` 的全部教学改造。
- **mock-api**（原 PR：`vanbanana/polystudio-platform#2`，分支 `devin/...-mock-provider-api`）：`mock-api/` 整个目录。

两条分支都基于原 `main`，分别只动 `frontend/` 和 `mock-api/`，合并无冲突。
